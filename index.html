Inference Engine

Cactus Compute v1.14

C++ inference engine compiled to ARM64. Zero-copy memory mapping, INT4 quantization, Apple Neural Engine acceleration. Same engine runs on phone, tablet, and laptop.

cactuscompute.com

Language Model

Gemma 4 E2B (Google DeepMind)

2.3B effective parameters, 5.1B total. Native audio input via 300M-parameter conformer encoder. Reasons directly over raw audio — no separate transcription step needed. Apache 2.0.

ai.google.dev/gemma

Speech-to-Text

NVIDIA Parakeet TDT 0.6B

4.65% WER — lowest error rate of any on-device speech model. Runs as a fast first pass before the LLM. Apple NPU accelerated for sub-second transcription.

Smart Glasses

Even Realities G2 + R1 Ring

576x288 px per eye, 4-bit greyscale micro-LED. Built-in mic (16kHz PCM S16LE mono). R1 ring for selection via BLE. Looks like regular glasses.

The Pipeline

01

Audio Capture

G2 glasses mic captures PCM audio (16kHz, 16-bit, mono) and streams it over BLE to the iPhone. The Even Realities app relays audio frames to our web app running in its WebView.

even_hub_sdk → audioEvent.audioPcm (Uint8Array)
02

Speech Recognition

3-second audio buffer is sent via HTTP POST to the Marginalia app's local server (localhost:8080). Parakeet TDT transcribes the audio to text with 95.35% accuracy.

cactusTranscribe(sttModel, pcmData) → "I've been feeling burned out..."
03

Tactical Analysis

Gemma 4 E2B receives the transcript with a system prompt containing the scenario context (role, relationship, organizational dynamics). It generates exactly 3 response options with optional function calls.

cactusComplete(llmModel, messages, tools) → {"options": [...]}
04

Lens Display

Options are rendered on the G2 lens via textContainerUpgrade — flicker-free in-place text updates. Ring rotation moves the selection indicator. Ring click triggers the function call.

bridge.textContainerUpgrade() → ◉ Approve ○ Counter ○ Defer
05

Function Execution

When the wearer taps the R1 ring, the selected option's function call executes on-device. For this demo: createCalendarHold(title, startDate, endDate). A calendar event appears on the iPhone screen.

POST /tool-call-json → {"status": "created", "event": {...}}
